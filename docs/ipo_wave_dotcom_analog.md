# 底層邏輯:IPO 超級年的歷史類比 — 會不會重蹈 dot-com?

> 主理人(2026-06):歷史上是否也有像今年這樣很多科技股新上市?他們後來怎麼樣?
> 會不會又重蹈 dot-com bubble?

## 歷史:IPO 洪峰常出現在循環「頂部附近」
- **1999–2000 dot-com**:IPO 狂熱(Pets.com/Webvan…),多數**在 2000–2002 崩盤中死掉**
  (Nasdaq −78%)。**少數倖存**(AMZN 一度 −95% 後百倍)。
- **2020–2021 SPAC/IPO 狂熱**:你正好經歷——很多你前面篩到的票(RKLB/ASTS/LUNR/PL…)
  是 2021 SPAC,**2022 被腰斬再腰斬**。
- **共同規律(為何 IPO 洪峰=晚期訊號)**:
  1. **內部人/VC 在高估值套現**(賣給市場 = 派發);
  2. **新股供給吸走流動性**(呼應你的「非2021、廣度小」——$3.6T 新供給會排擠現有股);
  3. **窗口開越久、品質越差**(好公司先上,後面越來越爛);
  4. **邊際買家是散戶亢奮**。

## 2026 會重演嗎?相同與不同
- **相同(該警惕)**:**晚期流動性排擠 + 亢奮**的風險一樣;若再疊**BTC 後段循環衝頂**
  (見 `thesis_crypto_cycle_2026.md`)+ 你的**九月變盤** → **多個晚期訊號正在匯聚**。
- **不同(別過度看空)**:2026 的巨頭(SpaceX/OpenAI/Stripe/Databricks)是**大型、有營收、
  真生意**,不是 1999 的無盈利概念股 → **品質高得多**,不是純 dot-com 翻版。
- ⇒ 結論:**不是「立刻崩」,而是「晚期」**。可以吃最後一段,但要把「**2027 級的回檔**」
  當成基準情境來規劃,而不是假設牛市無限。

## 操作含義(已內建在系統)
1. **燃料閘**:只買真賺錢/真題材的 IPO 代理;**爛 IPO/純敘事自動被 🚫/🪨 刷掉**——
   這正是避免「買到 Pets.com」的機制。
2. **別買 IPO 首日/別追噴出**:代理漲在預期、賣在 news;首日不接刀。
3. **留乾火藥到變盤**:IPO 洪峰+幣圈衝頂若同時做頭,現金是你的選擇權。
4. **倖存者思維**:dot-com 也有 AMZN——**核心壓有營收/護城河的**(MSFT/AMZN/V/MA/RKLB),
   投機壓 pre-profit(ASTS/礦工/山寨)且小倉。

## 一句話
**IPO 洪峰 + 幣圈後段循環 + 窄廣度 = 多個晚期訊號匯聚。** 規律上這像「牛市最後一段 + 之後
一個 2027 級回檔」。吃最後一段、用燃料閘挑質、留現金——**別當那個在頂部抱著爛 IPO 和山寨的人。**
