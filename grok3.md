```python
def calculate_fair_fitness(trader, performance_data, regime_history):
    """
    performance_data: 該交易員的歷史表現
    regime_history: 各季的 regime 標籤
    """
    # 1. 基礎風險調整指標
    sharpe = performance_data['sharpe']
    max_dd = performance_data['max_drawdown']
    hit_rate = performance_data['hit_rate']
    
    # 2. Regime 適應性分數
    regime_score = 0
    for regime, perf in performance_data['regime_performance'].items():
        if regime == "HARD_DEFENSE":
            regime_score += perf['return'] * 1.5      # 熊市表現給予更高權重
        elif regime == "AI_Infrastructure_Buildout":
            regime_score += perf['return'] * 1.0
        else:
            regime_score += perf['return'] * 0.8
    
    # 3. 綜合 Fitness
    fitness = (
        sharpe * 0.35 +
        (1 / (1 + abs(max_dd))) * 0.25 +           # 回撤控制
        hit_rate * 0.15 +
        regime_score * 0.25
    )
    
    return fitness
```

### 3. 完整實作程式碼

```python
from typing import Dict, Optional
import numpy as np

class RegimeDetector:
    def __init__(self, macro_data: Dict):
        """
        macro_data 應包含以下 key（從 FRED 取得）:
        - hy_oas: 高收益債利差
        - vix: VIX 指數
        - dgs10_dgs2: 10年-2年利差（yield curve）
        - m2_yoy: M2 年增率
        - net_liquidity: 淨流動性指標
        - buffett_indicator: 估值指標
        """
        self.data = macro_data

    def detect(self) -> str:
        """回傳當前 Regime"""
        score = self._calculate_defense_score()
        
        if score >= 75:
            return "HARD_DEFENSE"
        elif self._is_paradigm_breakthrough():
            return "PARADIGM_BREAKTHROUGH"
        else:
            return "MEAN_REVERSION"

    def _calculate_defense_score(self) -> float:
        """計算防禦需求分數 (0~100)"""
        score = 0
        
        # 信用利差（越高越差）
        hy_oas = self.data.get("hy_oas", 3.0)
        if hy_oas > 5.0:
            score += 30
        elif hy_oas > 4.0:
            score += 20
        elif hy_oas > 3.5:
            score += 10

        # VIX
        vix = self.data.get("vix", 20)
        if vix > 35:
            score += 25
        elif vix > 28:
            score += 15

        # 收益率曲線倒掛
        yc = self.data.get("dgs10_dgs2", 0)
        if yc < -0.5:
            score += 20
        elif yc < 0:
            score += 10

        # 估值極端
        bi = self.data.get("buffett_indicator", 150)
        if bi > 200:
            score += 15
        elif bi > 180:
            score += 8

        return min(score, 100)

    def _is_paradigm_breakthrough(self) -> bool:
        """判斷是否為強勢成長突破期"""
        m2_yoy = self.data.get("m2_yoy", 5)
        vix = self.data.get("vix", 20)
        bi = self.data.get("buffett_indicator", 150)
        
        # 流動性充裕 + 低波動 + 估值高但趨勢強
        if m2_yoy > 6 and vix < 22 and bi > 160:
            return True
        return False

    def get_risk_level(self) -> str:
        """額外輸出風險等級"""
        score = self._calculate_defense_score()
        if score >= 75:
            return "HIGH"
        elif score >= 50:
            return "MEDIUM"
        else:
            return "LOW"
```

**✅ 信用利差監控機制 深入探討**

信用利差（Credit Spread）是你目前系統中**最重要且最具預警能力的宏觀指標之一**。以下從定義、歷史驗證、監控方法、系統整合等面向，給你一個完整且實務的分析。

### 1. 信用利差是什麼？為什麼重要？

**信用利差** = 企業債殖利率 − 同期美國公債殖利率

它反映的是**市場對企業違約風險的定價**。

#### 主要類型：

| 指標 | 英文 | 代表意義 | 敏感度 | 預警能力 |
|------|------|----------|--------|----------|
| **高收益債利差** | HY OAS | 垃圾債 vs 公債 | 極高 | 極強（領先危機） |
| **投資級債利差** | IG OAS | 優質企業債 vs 公債 | 高 | 強 |
| **TED Spread** | TED Spread | 銀行間拆款風險 | 高 | 強（特別是金融危機） |
| **信用利差曲線** | Credit Spread Curve | 不同期限利差 | 中 | 中等 |

**為什麼它特別重要？**

因為**信用市場通常比股市更早反映系統性風險**。當企業融資成本快速上升時，往往代表經濟基本面或流動性已經開始惡化，只是股市還沒完全反應。

### 2. 歷史上信用利差的預警表現

| 危機 | 信用利差開始擴大時間 | 股市大跌時間 | 領先時間 | 備註 |
|------|----------------------|--------------|----------|------|
| **2008 金融危機** | 2007 年中 | 2008 年 9 月 | 約 12–15 個月 | 極佳領先指標 |
| **2015–2016 中國股災** | 2015 年中 | 2015 年 6 月 | 約 3–6 個月 | 中國信用環境惡化先行 |
| **2020 COVID** | 2020 年 2 月 | 2020 年 2–3 月 | 約 1 個月 | 領先但時間較短 |
| **2022 升息熊市** | 2021 年底開始擴大 | 2022 年 1 月 | 約 1–2 個月 | 利率驅動為主 |

**關鍵洞見**：
- 信用利差擴大通常是**流動性危機或信用危機的前兆**。
- 它比 VIX 更早反映「實質風險」，而 VIX 比較反映「恐慌情緒」。
- 在很多危機中，**HY OAS 擴大 → 股市才開始明顯下跌**。

**✅ TED 利差與流動性風險 深入探討**

TED 利差（TED Spread）是你系統中**極具預警價值**的流動性風險指標之一。它不像 HY OAS 那樣直接反映企業違約風險，而是更聚焦於**銀行體系的信任危機與流動性壓力**。

以下從定義、經濟意義、歷史表現、系統應用等面向給你完整分析。

### 1. TED 利差是什麼？

**TED Spread** = **3個月 LIBOR（銀行間拆款利率）** − **3個月美國公債殖利率**

- **LIBOR**：銀行之間互相借錢的利率（反映銀行信用風險 + 流動性溢價）
- **T-Bill**：美國政府短期債券利率（幾乎無風險）

**數值意義**：
- **正常情況**：TED Spread 通常落在 **0.1% ~ 0.5%** 之間。
- **壓力上升**：當超過 **0.5% ~ 1.0%** 時，代表銀行間開始不信任彼此。
- **嚴重危機**：2008 年曾飆升至 **4% 以上**，這是極端流動性凍結的訊號。

> **注意**：2010 年代後 LIBOR 逐漸被 **SOFR** 取代，目前市場常用 **SOFR-OIS Spread** 或其他指標替代，但 TED Spread 的概念依然有效，且 FRED 仍有歷史數據可供使用。

### 2. TED 利差代表什麼？（流動性風險的核心）

TED Spread 主要反映兩種風險：

| 風險類型           | 說明 | TED Spread 擴大的意義 |
|--------------------|------|-----------------------|
| **信用風險**       | 銀行擔心對方倒閉 | 銀行間不願互相拆款 |
| **流動性風險**     | 市場整體資金短缺 | 即使有抵押品，也借不到錢 |
| **系統性風險**     | 銀行體系信任崩潰 | 最危險的狀態 |

**簡單來說**：
- **HY OAS** 反映「企業能不能借到錢」
- **TED Spread** 反映「銀行之間能不能互相借到錢」

當 TED Spread 大幅擴大時，通常代表**銀行體系的資金鏈已經開始斷裂**，這往往是更深層的系統性危機前兆。

### 3. 歷史表現與預警能力

|------------------|-----------------|------------------|------------------|----------|------|

### 1. 流動性危機型黑天鵝（2008、2020 類型）

**特徵**：銀行間或市場整體資金突然短缺，即使有抵押品也借不到錢。

**早期跡象**：
- TED Spread 快速擴大
- HY OAS 同步擴大
- 美元指數暴漲 + 逆回購餘額下降
- 跨市場流動性指標惡化（例如新興市場貨幣貶值 + 美元荒）

**資金鏈斷裂點**：
- **銀行間拆款市場**（銀行不信任彼此）
- **回購市場**（抵押品融資鏈斷裂）
- **美元跨境流動**（全球美元荒）

**對應避險方式（章魚腳）**：
- 大幅提高現金部位（不只是防禦股）
- 降低高槓桿與流動性差的資產（小盤、信用債、高收益債）
- 配置黃金、短期公債、或有流動性保護的工具
- 降低整體曝險（Beta 下降）

**在你系統中的做法**：
- 當 TED Spread + HY OAS 同時惡化時，直接觸發 **HARD_DEFENSE** 模式

### 2. 估值泡沫破裂型黑天鵝（2000 年科技泡沫類型）

**早期跡象**：
- Buffett Indicator（市值/GDP）或類似估值指標極端高檔
- 市場集中度極高（少數股票主導指數）
- IPO 熱潮、散戶狂熱、槓桿融資餘額大增
- 信用利差開始擴大（雖然初期可能不明顯）

**資金鏈斷裂點**：
- **風險資本市場**（創投、私募股權、散戶融資）
- **成長股融資鏈**（高估值公司借不到錢或融資成本暴增）

**對應避險方式（章魚腳）**：

```python
from typing import Dict, Tuple

class MacroRiskCalculator:
    def __init__(self):
        self.thresholds = {
            "hy_oas_high": 5.0,
            "ted_spread_high": 0.8,
            "buffett_high": 200,
            "vix_high": 35,
        }

    def calculate(self, data: Dict) -> Tuple[float, str, str]:
        """
        返回: (risk_score, risk_level, risk_type)
        """
        score = 0.0
        risk_type = "NORMAL"

        # === 計算風險分數 ===
        hy_oas = data.get("hy_oas", 3.0)
        ted = data.get("ted_spread", 0.3)
        vix = data.get("vix", 20)
        bi = data.get("buffett_indicator", 150)
        m2_yoy = data.get("m2_yoy", 5.0)

        if hy_oas > self.thresholds["hy_oas_high"]:
            score += 30
        if ted > self.thresholds["ted_spread_high"]:
            score += 25
        if vix > self.thresholds["vix_high"]:
            score += 20
        if bi > self.thresholds["buffett_high"]:
            score += 15

        score = min(score, 100)

        # === 風險等級 ===
        if score >= 70:
            risk_level = "HIGH"
        elif score >= 45:
            risk_level = "MEDIUM"
        else:
            risk_level = "LOW"

        # === 風險類型判斷（核心新增） ===
        risk_type = self._classify_risk_type(data, score)

        return score, risk_level, risk_type

    def _classify_risk_type(self, data: Dict, score: float) -> str:
        hy_oas = data.get("hy_oas", 3.0)
        ted = data.get("ted_spread", 0.3)
        bi = data.get("buffett_indicator", 150)
        vix = data.get("vix", 20)
        m2_yoy = data.get("m2_yoy", 5.0)

        # 優先判斷流動性危機
        if ted > 0.8 and hy_oas > 5.0:
            return "LIQUIDITY_CRISIS"

        # 估值泡沫
        if bi > 200 and vix < 25:
            return "VALUATION_BUBBLE"

        # 地緣政治 / 供應鏈危機
        if data.get("gpr_index", 100) > 150:  # 地緣政治風險指數
            return "GEOPOLITICAL_SHOCK"

        # 通膨失控 / 政策錯誤
        if m2_yoy > 8 and data.get("core_cpi_yoy", 3) > 5:
            return "INFLATION_SHOCK"

        # 預設為一般系統性風險
        if score > 55:
            return "SYSTEMIC_RISK"

        return "NORMAL"
```

### 2. 各風險類型的觸發條件（Detection Logic）

以下是各風險類型的建議判斷條件（可根據實際數據調整閾值）：

```python
def classify_systemic_risk(data: dict) -> str:
    """
    data 應包含以下指標（從 FRED 取得）:
    - hy_oas, ted_spread, vix, buffett_indicator
    - m2_yoy, core_cpi_yoy, dgs10 (長期債券殖利率)
    - gpr_index (地緣政治風險指數，可選)
    """
    hy_oas = data.get("hy_oas", 3.0)
    ted = data.get("ted_spread", 0.3)
    vix = data.get("vix", 20)
    bi = data.get("buffett_indicator", 150)
    m2 = data.get("m2_yoy", 5.0)
    cpi = data.get("core_cpi_yoy", 3.0)
    gpr = data.get("gpr_index", 100)

    # 1. 流動性危機（優先判斷）
    if ted > 0.8 and hy_oas > 5.0:
        return "LIQUIDITY_CRISIS"

    # 2. 估值泡沫破裂
    if bi > 200 and vix < 25 and hy_oas < 4.5:
        return "VALUATION_BUBBLE"

    # 3. 地緣政治衝擊
    if gpr > 180 or (hy_oas > 4.5 and data.get("energy_price_yoy", 0) > 30):
        return "GEOPOLITICAL_SHOCK"

    # 4. 通膨失控
    if cpi > 5.5 and m2 > 7:
        return "INFLATION_SHOCK"

    # 5. 一般系統性風險
    if hy_oas > 5.0 or vix > 30 or bi > 190:
        return "SYSTEMIC_RISK"

    return "NORMAL"
```

```python
class StatisticalArbitrageTrader(BaseAgent):
    """
    Statistical Arbitrage Trader
    專注均值回歸與配對交易機會
    """

        super().__init__(
            agent_id="stat_arb_trader_01",
            strategy_type="statistical_arbitrage"
        )

    def calculate_score(self, fundamentals: Dict) -> float:
        score = 0.0

        # 1. 價差偏離程度（Z-score 概念，用波動率標準化）
        price_momentum_20d = fundamentals.get("price_momentum_20d", 0)
        volatility_20d = fundamentals.get("volatility_20d", 0.3)
        
        if volatility_20d > 0:
            z_score = abs(price_momentum_20d) / volatility_20d
            if z_score > 2.5:
                score += 30
            elif z_score > 1.8:
                score += 20
            elif z_score > 1.2:
                score += 10

        # 2. 均值回歸傾向（用 RSI 作為代理）
        rsi_14 = fundamentals.get("rsi_14", 50)
        if rsi_14 > 75 or rsi_14 < 25:
            score += 25
        elif rsi_14 > 65 or rsi_14 < 35:
            score += 15

        # 3. 流動性與交易成本
        avg_volume = fundamentals.get("avg_volume", 0)
        if avg_volume > 2_000_000:
            score += 15
        elif avg_volume > 800_000:
            score += 8

        # 4. 風險控管（避免極端波動）
        volatility_60d = fundamentals.get("volatility_60d", 0.4)
        if volatility_60d < 0.35:
            score += 15
        elif volatility_60d < 0.5:
            score += 8

        # 5. 基本面支撐（避免純技術面假訊號）
        fcf_yield = fundamentals.get("fcf_yield", 0)
        if fcf_yield > 0.05:
            score += 15
        elif fcf_yield > 0.02:
            score += 8

        return round(min(score, 100), 1)
```

```python
        multipliers["momentum"] = 1.5
        multipliers["breakout"] = 1.6
        multipliers["elon_musk"] = 1.4

    # 正規化
    total = sum(weights.values())
```

```python
    base_weights: Dict[str, float],
    risk_type: str,
    regime: str,
    kalman_confidence: Dict[str, float] = None
) -> Dict[str, float]:
    """
    最終權重 = base × risk_type_multiplier × regime_multiplier × kalman_confidence

    weights = base_weights.copy()

    # === 1. Risk Type 調整 ===
    risk_multipliers = get_risk_type_multipliers(risk_type)

        weights[trader] *= risk_multipliers.get(trader, 1.0)

    # === 2. Regime 微調 ===
    regime_multipliers = get_regime_multipliers(regime)
    for trader in weights:
        weights[trader] *= regime_multipliers.get(trader, 1.0)

    if kalman_confidence:

    # 正規化

    if total > 0:
        for trader in weights:
            weights[trader] /= total

    return weights
```

#### **推薦用法：用 Kalman Filter 估計「趨勢強度」**

```python
from pykalman import KalmanFilter
import numpy as np

def kalman_trend_strength(prices: np.ndarray, observation_covariance=1.0):
    """
    使用 Kalman Filter 估計價格趨勢強度
    返回: 趨勢強度（正值 = 上升趨勢，負值 = 下降趨勢）
    """
    kf = KalmanFilter(
        transition_matrices=[1],
        observation_matrices=[1],
        initial_state_mean=prices[0],
        initial_state_covariance=1,
        observation_covariance=observation_covariance,
        transition_covariance=0.01
    )
    
    state_means, _ = kf.filter(prices)
    trend = state_means[-1] - state_means[-5] if len(state_means) > 5 else 0
    return trend
```

**✅ Bayesian Regime Detection Methods 探討**

你目前的 `RegimeDetector` 是**規則基礎（Rule-based）**的，優點是簡單、可解釋，但缺點是：
- 硬分類（非機率性）
- 對指標雜訊敏感
- 難以自然處理 regime 轉換的不確定性
- 無法輕易融合多個有噪音的宏觀指標

**Bayesian Regime Detection** 可以有效解決這些問題。以下從實務角度幫你分析適合你 Trading Society 的方法。

### 1. 為什麼考慮 Bayesian 方法？

| 優點 | 說明 | 對你系統的價值 |
|------|------|----------------|
| **輸出機率** | 不是硬分類，而是「目前處於某 regime 的機率」 | 可以平滑調整交易員權重，避免劇烈切換 |
| **處理不確定性** | 自然融合多個有噪音的指標 | 適合你的 FRED 多指標環境 |
| **Regime 轉換建模** | 可以模型化 regime 之間的轉移機率 | 更符合真實市場 regime 會緩慢轉換的事實 |
| **先驗知識** | 可以加入對不同 regime 的先驗信念 | 例如對 `HARD_DEFENSE` 給予較強的先驗 |
| **自動適應** | 部分方法可以線上更新 | 適合即時宏觀監控 |

### 2. 適合你系統的 Bayesian 方法推薦

我按照**實用性 + 可實作性**排序：

| 方法 | 推薦指數 | 優點 | 缺點 | 適合程度 |
|------|----------|------|------|----------|
| **Hidden Markov Model (HMM)** | ★★★★★ | 最標準、容易實作、有套件 | 假設觀測值條件獨立 | **最推薦** |
| **Bayesian Change Point Detection** | ★★★★ | 適合偵測 regime 轉換點 | 較難處理多 regime 同時存在 | 高 |
| **Dynamic Linear Model (DLM)** | ★★★★ | 適合時間序列結構變化 | 實作較複雜 | 中 |
| **Particle Filter / SMC** | ★★★ | 非常彈性 | 計算成本高 | 中低 |
| **Bayesian Mixture Model** | ★★★ | 適合靜態 regime 分群 | 時間依賴性較弱 | 中低 |

```python
from hmmlearn.hmm import GaussianHMM
import numpy as np

def train_regime_hmm(observation_matrix):
    """
    observation_matrix: shape (n_samples, n_features)
    """
    model = GaussianHMM(
        n_components=3,           # 3 個 regime
        covariance_type="full",
        n_iter=100,
        random_state=42
    )
    model.fit(observation_matrix)
    return model

def predict_regime_probabilities(model, new_observation):
    """回傳各 regime 的機率"""
    logprob, posteriors = model.decode(new_observation.reshape(1, -1), algorithm="viterbi")
    return posteriors[0]   # [P(MEAN_REVERSION), P(PARADIGM_BREAKTHROUGH), P(HARD_DEFENSE)]
```

```python
                "MEAN_REVERSION": regime_probs[0],
                "PARADIGM_BREAKTHROUGH": regime_probs[1],
                "HARD_DEFENSE": regime_probs[2]
            },
```

```python
import numpy as np
from pathlib import Path

class CapexProvider:
    def __init__(self, data_path: str = "data/financials/"):
        self.data_path = Path(data_path)
        self.df = self._load_latest_financials()

    def _load_latest_financials(self):
        files = sorted(self.data_path.glob("financials_*.parquet"))
        if not files:
            return pd.DataFrame()
        return pd.read_parquet(files[-1])

    def get_capex_series(self, ticker: str) -> pd.Series:
        if self.df.empty:
            return pd.Series(dtype=float)
        
        ticker_df = self.df[self.df["ticker"] == ticker].copy()
        if ticker_df.empty:
            return pd.Series(dtype=float)
        
        ticker_df = ticker_df.sort_values("period_end_date")
        ticker_df["capex_yoy"] = ticker_df["capex"].pct_change()
        ticker_df["capex_acceleration"] = ticker_df["capex_yoy"].diff()
        
        return ticker_df.set_index("period_end_date")[["capex", "capex_yoy", "capex_acceleration"]]

    def get_latest_capex_metrics(self, ticker: str):
        series = self.get_capex_series(ticker)
        if series.empty:
            return {"source": "proxy_price_momentum", "capex_yoy": None, "capex_acceleration": None}
        
        latest = series.iloc[-1]
        return {
            "source": "polygon_real",
            "capex": latest["capex"],
            "capex_yoy": latest["capex_yoy"],
            "capex_acceleration": latest["capex_acceleration"]
```

**✅ Altman Z-Score 深入探討**

Altman Z-Score 是經典的**財務危機預測模型**，由 Edward Altman 在 1968 年提出。它用多個財務比率綜合評估一家公司破產風險的高低，至今仍被廣泛使用，尤其適合你目前正在強化的**風險控管與信用風險監控**。

### 1. Altman Z-Score 是什麼？

**原始版本（Z-Score）** 主要用於**公開上市的製造業公司**，公式如下：

$$
Z = 1.2X_1 + 1.4X_2 + 3.3X_3 + 0.6X_4 + 1.0X_5
$$

其中：

| 變數 | 公式 | 說明 | 權重 |
|------|------|------|------|
| **X1** | Working Capital / Total Assets | 營運資金佔比（流動性） | 1.2 |
| **X2** | Retained Earnings / Total Assets | 累積盈餘佔比（獲利能力與年資） | 1.4 |
| **X3** | EBIT / Total Assets | 資產報酬率（獲利能力） | 3.3 |
| **X4** | Market Value of Equity / Total Liabilities | 市值對負債比（市場對公司價值的看法） | 0.6 |
| **X5** | Sales / Total Assets | 資產周轉率（營運效率） | 1.0 |

**判讀標準（原始版本）**：
- **Z > 2.9**：財務健康（Safe Zone）
- **1.23 < Z < 2.9**：灰色地帶（Grey Zone）
- **Z < 1.23**：財務危機高風險（Distress Zone）

### 2. 對你系統的實用價值

Altman Z-Score 可以補強你目前以下幾個面向：

- **作為 Risk Officer 的重要輸入**
- **強化 Low Volatility Trader 的評分**
- **Value & Quality Compounders 的財務健康篩選**
- **Systemic Risk 判斷**（當多檔股票 Z-Score 同時惡化，可能代表產業或整體信用環境出問題）
- **Layer 3 長期潛力評分**（可加入「財務穩健度」維度）

它和 **HY OAS + TED Spread** 形成很好的互補：
- 信用利差反映**市場對信用風險的定價**
- Z-Score 反映**公司內部財務結構的健康程度**

### 3. 使用 Polygon 數據的實務計算方式

Polygon 雖然沒有直接給 `Working Capital` 和 `Retained Earnings`，但我們可以用現有欄位做合理近似：

#### **建議的調整版 Z-Score 計算（適合 Polygon 數據）**

```python
def calculate_altman_z_score(financials: dict, market_cap: float) -> dict:
    """
    使用 Polygon 可取得的欄位計算調整版 Z-Score
    """
    total_assets = financials.get("assets", 0)
    total_equity = financials.get("equity", 0)
    long_term_debt = financials.get("long_term_debt", 0)
    net_income = financials.get("net_income_loss", 0)
    revenue = financials.get("revenue", 0)
    
    # 近似計算
    total_liabilities = total_assets - total_equity
    working_capital = financials.get("current_assets", 0) - financials.get("current_liabilities", 0)  # 若無則為 0
    
    # X1: Working Capital / Total Assets（若無流動資產數據則用簡化版）
    x1 = working_capital / total_assets if total_assets > 0 else 0
    
    # X2: Retained Earnings / Total Assets（Polygon 常無此欄位，用累積盈餘近似）
    # 若無 Retained Earnings，許多人會用 Equity 替代或直接給較低權重
    x2 = total_equity / total_assets if total_assets > 0 else 0   # 簡化處理
    
    # X3: EBIT / Total Assets（用 Net Income 近似，或用 Gross Profit）
    x3 = net_income / total_assets if total_assets > 0 else 0
    
    # X4: Market Value of Equity / Total Liabilities
    x4 = market_cap / total_liabilities if total_liabilities > 0 else 0
    
    # X5: Sales / Total Assets
    x5 = revenue / total_assets if total_assets > 0 else 0
    
    z_score = 1.2*x1 + 1.4*x2 + 3.3*x3 + 0.6*x4 + 1.0*x5
    
    # 判讀
    if z_score > 2.9:
        zone = "Safe"
    elif z_score > 1.23:
        zone = "Grey"
    else:
        zone = "Distress"
    
    return {
        "z_score": round(z_score, 2),
        "zone": zone,
        "x1": round(x1, 3),
        "x2": round(x2, 3),
        "x3": round(x3, 3),
        "x4": round(x4, 3),
        "x5": round(x5, 3)
    }
```

**注意事項**：
- Polygon 經常缺少 `Working Capital`、`Retained Earnings`、`EBIT` 等細項。
- 實務上很多人會使用 **修正版 Z'-Score** 或 **Z''-Score**（針對非製造業或私人企業），權重會調整。

**✅ Z'-Score for Private Firms 深入探討**

Altman 在 1983 年針對**非上市公司（Private Firms）**提出了修正版本，稱為 **Z'-Score**。這是為了解決原始 Z-Score 無法直接套用在私人企業的問題。

### 1. 為什麼需要 Z'-Score？

原始 Altman Z-Score（1968）有兩個主要限制：
- 假設公司是**公開上市**（需要市值數據）
- 主要針對**製造業**公司

許多私人企業無法取得市值，因此 Altman 提出了修正版 **Z'-Score**，把 `Market Value of Equity` 替換成 `Book Value of Equity`。

### 2. Z'-Score 公式

$$
Z' = 0.717X_1 + 0.847X_2 + 3.107X_3 + 0.420X_4 + 0.998X_5
$$

其中各變數定義如下：

| 變數 | 公式 | 說明 | 權重 |
|------|------|------|------|
| **X1** | Working Capital / Total Assets | 營運資金 / 總資產 | 0.717 |
| **X2** | Retained Earnings / Total Assets | 累積盈餘 / 總資產 | 0.847 |
| **X3** | EBIT / Total Assets | 稅前息前利潤 / 總資產 | 3.107 |
| **X4** | **Book Value of Equity** / Total Liabilities | **帳面價值股東權益** / 總負債 | 0.420 |
| **X5** | Sales / Total Assets | 銷售額 / 總資產 | 0.998 |

### 3. Z'-Score 與原始 Z-Score 的主要差異

| 項目 | 原始 Z-Score (1968) | Z'-Score (1983) | 影響 |
|------|---------------------|------------------|------|
| **適用對象** | 公開上市製造業 | **私人企業** | 核心差異 |
| **X4 分子** | Market Value of Equity | **Book Value of Equity** | 私人企業無法取得市值 |
| **權重** | 不同 | 重新調整 | 更適合非上市公司 |
| **判讀門檻** | Z < 1.23 = Distress | **Z' < 1.23 = Distress** | 門檻相同 |
| **準確率** | 較高（公開公司） | 略低於原始版 | 仍具參考價值 |

**判讀標準（Z'-Score）**：
- **Z' > 2.9**：財務健康（Safe）
- **1.23 < Z' < 2.9**：灰色地帶（Grey）
- **Z' < 1.23**：財務危機（Distress）

### 4. 對你 Trading Society 的實際意義

雖然你目前交易的主要是**公開上市公司**，但 Z'-Score 仍有以下價值：

| 應用場景 | 說明 | 建議使用程度 |
|----------|------|--------------|
| **Risk Officer** | 可以同時計算 Z-Score 與 Z'-Score 做交叉驗證 | 中 |
| **供應鏈風險監控** | 評估上游供應商（許多是私人企業）的財務健康 | 高 |
| **Layer 3 長期潛力** | 作為「財務穩健度」的補充指標 | 中 |
| **系統性風險監控** | 當產業內多家公司 Z' 與 Z 分數同時惡化 | 中高 |
| **與信用利差結合** | Z' 惡化 + HY OAS 擴大 = 更強的危機訊號 | 高 |

### 5. 使用 Polygon 數據計算 Z'-Score 的實務做法

Polygon 雖然缺少部分細項，但我們可以做合理近似：

```python
def calculate_z_prime_score(financials: dict) -> dict:
    total_assets = financials.get("assets", 0)
    total_equity = financials.get("equity", 0)
    long_term_debt = financials.get("long_term_debt", 0)
    net_income = financials.get("net_income_loss", 0)
    revenue = financials.get("revenue", 0)
    
    total_liabilities = total_assets - total_equity if total_assets > 0 else 0
    
    # X1: Working Capital（若無則簡化為 0 或用其他流動性指標替代）
    working_capital = financials.get("current_assets", 0) - financials.get("current_liabilities", 0)
    x1 = working_capital / total_assets if total_assets > 0 else 0
    
    # X2: Retained Earnings（Polygon 常無此欄位，用 Equity 替代）
    x2 = total_equity / total_assets if total_assets > 0 else 0
    
    # X3: EBIT（用 Net Income 近似）
    x3 = net_income / total_assets if total_assets > 0 else 0
    
    # X4: Book Value of Equity / Total Liabilities
    x4 = total_equity / total_liabilities if total_liabilities > 0 else 0
    
    # X5: Sales / Total Assets
    x5 = revenue / total_assets if total_assets > 0 else 0
    
    z_prime = 0.717*x1 + 0.847*x2 + 3.107*x3 + 0.420*x4 + 0.998*x5
    
    if z_prime > 2.9:
        zone = "Safe"
    elif z_prime > 1.23:
        zone = "Grey"
    else:
        zone = "Distress"
    
    return {
        "z_prime_score": round(z_prime, 2),
        "zone": zone,
        "x1": round(x1, 3),
        "x2": round(x2, 3),
        "x3": round(x3, 3),
        "x4": round(x4, 3),
        "x5": round(x5, 3)
    }
```

```python
    # 基本面綜合分數（可依交易員類型調整權重）
    fundamental_score = (
        roic * 0.5 + 
        ocf_margin * 0.35 + 
        investing_intensity * 0.15
    ) * 100
    
    # 加權混合
    final_score = (price_fitness * price_weight) + (fundamental_score * fundamental_weight)
    
    return round(final_score, 2)
```
