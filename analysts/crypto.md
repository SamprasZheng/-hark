**x402** 是由 Coinbase 所開發的開源支付協定，它透過復刻並應用長期閒置的 **HTTP 402「Payment Required」（需要付款）狀態碼**，讓網路應用程式與 AI 代理能夠直接透過標準 HTTP 協定進行即時的穩定幣支付。

### x402 協定的運作流程

x402 將支付功能原生地整合到網路互動中，其核心運作流程如下：

1. **發起請求**：客戶端（例如 AI 代理）向伺服器請求存取 API 數據或數位資源。
2. **伺服器回傳 HTTP 402 錯誤**：如果該請求未包含付款證明，伺服器會拒絕存取，並回傳 HTTP 402 狀態碼。同時，伺服器會在 JSON 負載（Payload）中提供付款明細，包含所需金額、賣方錢包地址、代幣合約（如 USDC）以及區塊鏈網路等資訊。
3. **提交加密簽名與付款**：AI 代理讀取這些指令後，會自動透過其加密錢包產生一組數位簽名授權付款（遵循 EIP-712 標準），接著將包含付款資訊的 `X-PAYMENT` 標頭附加到請求中，重新發送給伺服器。
4. **驗證與即時結算**：伺服器（或透過第三方中繼伺服器 Facilitator）驗證付款憑證。一旦確認有效，伺服器便會回傳 AI 代理所請求的資源，並將交易上鏈結算（例如在 Base 網路上結算僅需約 200 毫秒）。

### x402 如何賦予 AI 代理自主交易能力？

傳統的支付系統（如信用卡、銀行轉帳）是為「人類」設計的，充滿了手動流程、帳戶註冊、訂閱制和高昂的手續費，這些都成為了 AI 系統自動化的瓶頸。x402 透過以下方式徹底改變了這一點：

* **消除人類介入與 API 密鑰管理**：AI 代理不再需要人類預先綁定信用卡、購買預付點數或設定 API 密鑰。只要 AI 代理配置了具備資金的錢包，就能在遇到 402 付費牆時**自主辨識金額並完成支付**，實現完全無摩擦的「機器對機器（M2M）」原生支付。
* **解鎖超微型支付（Micropayments）**：由於傳統信用卡手續費過高（通常有 0.30 美元的低消），微型支付難以實現。x402 透過區塊鏈基礎設施，使單筆交易手續費趨近於零，AI 代理可以進行低至 0.001 美元的極小額支付。這使得真正的**「按次 / 按使用量計費」（Pay-per-use）**成為可能。

### AI 代理的自主交易應用場景

有了 x402 的經濟自主權，AI 代理可以主動化身為經濟參與者，執行以下任務：

* **即時獲取上下文與數據**：當 AI 在進行研究或推理時，可以自主支付幾美分的微薄費用，突破新聞網站的付費牆來抓取文章，或存取即時的金融市場數據庫。
* **動態擴充運算資源**：AI 系統可以根據當下的任務需求，即時為自己購買額外的雲端儲存空間，或是以「每分鐘 0.50 美元」為單位租用 GPU 算力。
* **向其他 AI 購買服務**：AI 代理可以直接向其他專門的 AI 模型（如電腦視覺或語音生成 API）支付單次推理費用，自主協商並整合外部資源來完成目標。
